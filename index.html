浮点型

分类：float，double，long double，long long double
表示：
先将小数转换为二进制：整数部分同整数，小数部分：乘2取整法
浮点数分为三部分：符号位、尾数、指数
下文来自CDSN：
根据国际标准IEEE（电气和电子工程协会）规定，任何一个浮点数NUM的二进制数可以写为：
$NUM = (-1) ^ S * M * 2 ^ E$;//(S表示符号，E表示阶乘，M表示有效数字)
①当S为0时，表示一个正数；当S为1时，表示一个负数
②M表示有效数字，1<= M <2
③2^E表示指数
比如十进制的3.0，二进制就是0011.0 就可以写成（-1）^ 0 * 1.1 * 2 ^ 1
在比如十进制的-3.0，二进制就是-0011.0 就可以写成（-1）^ 1 * 1.1 * 2 ^ 1
而规定float类型有一个符号位（S），有8个指数位（E），和23个有效数字位（M）
double类型有一个符号位（S），有11个指数位（E），和52个有效数字位（M）
IEEE对于（有效数字）M和（指数）E有特殊的规定： （以float为例）
1.因为M的值一定是1<= M <2，所以它绝对可以写成1.xxxxxxx的形式，所以规定M在存储时舍去第一个1，只存储小数点之后的数字。这样做节省了空间，以float类型为例，就可以保存23位小数信息，加上舍去的1就可以用23位来表示24个有效的信息。
2.对于E（指数）E是一个无符号整数所以E的取值范围为（0~ 255），但是在计数中指数是可以为负的，所以规定在存入E时，在它原本的值上加上中间数（127），在使用时减去中间数（127），这样E的真正取值范围就成了（-127~128）。
对于E还分为三种情况：
①E不全为0，不全为1:
这时就用正常的计算规则，E的真实值就是E的字面值减去127（中间值)，M的值要加上最前面的省去的1。
②E全为0
这时指数E等于1-127为真实值，M不在加上舍去的1，而是还原为0.xxxxxxxx小数。这样为了表示0，和一些很小的整数。
所以在进行浮点数与0的比较时，要注意。
③E全为1
当M全为0时，表示±无穷大（取决于符号位）；当M不全为1时，表示这数不是一个数（NaN）
看不懂，了解一下，开心就好
根据储存原理可知，浮点数储存有误差（尤其是无限循环小数等）

数的机器码表示

机器数和真值

最高位为符号位（0为正，1为负），后加数值的二进制表示，叫做机器数
度娘：原码、补码、反码都是机器数
机器数的数值叫做真值
eg：$(5){10}$的8位机器数：00000101 真值：**+** 0000101
$(-5){10}$的8位机器数：10000101 真值：**-** 0000101

原码

最高位为符号位（0为正，1为负），后加数值的二进制表示，叫做原码
eg：$(5){10}$的8位原码：00000101
$(-5){10}$的8位原码：10000101
0的原码有两个：00000000、10000000

反码

正数的反码与原码相同
负数的反码即原码的符号位不变，其余位按位取反
eg：$(5){10}$的8位反码：00000101
$(-5){10}$的8位反码：11111010
0的反码有两个：00000000、11111111

补码

正数的补码与原码相同
负数的补码即反码加1
eg：$(5){10}$的8位补码：00000101
$(-5){10}$的8位补码：11111011
0的补码只有一个：00000000
整数的储存采用补码
意义：将加法转化为减法

来源：力扣（LeetCode）

题目描述

给你一个 n 行 m 列的矩阵，最开始的时候，每个单元格中的值都是 0。
另有一个索引数组 indices，indices[i] = [ri, ci] 中的 ri 和 ci 分别表示指定的行和列（从 0 开始编号）。
你需要将每对 [ri, ci] 指定的行和列上的所有单元格的值加 1。
请你在执行完所有 indices 指定的增量操作后，返回矩阵中 「奇数值单元格」 的数目。

样例

示例1

输入：n = 2, m = 3, indices = [[0,1],[1,1]]
输出：6
解释：最开始的矩阵是 [[0,0,0],[0,0,0]]。
第一次增量操作后得到 [[1,2,1],[0,1,0]]。
最后的矩阵是 [[1,3,1],[1,3,1]]，里面有 6 个奇数。

示例2

输入：n = 2, m = 2, indices = [[1,1],[0,0]]
输出：0
解释：最后的矩阵是 [[2,2],[2,2]]，里面没有奇数。

数据范围

1 <= n <= 50
1 <= m <= 50
1 <= indices.length <= 100
0 <= indices[i][0] < n
0 <= indices[i][1] < m
